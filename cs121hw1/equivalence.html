Equivalent. Since theta is only attributes from r, it does not matter the order of the natural join and the selecting. If the natural join is done first, a relation is made with values in the attributes of theta either from s or nulls. These values are then selected. If the select is done first, the attributes of theta are chosen and the natural join either adds the values from s or null values. Problem 1e

Not Equivalent. The left equation will give empty relations while the right value will give all the attributes of r with null values. Say r has some unique attributes, r and s have some shared attributes, and s has some unique attributes. The left equation natural join will contain attributes only contained in r, so the select (which chooses unique attributes from s) will select nothing. The right equation will select values only in s, so they will not appear in the natural join. Thus, the output will just be r with additional null values.
